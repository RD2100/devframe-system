# 用途说明：技术说明 v1.0

## 定位

本文可改写为一份技术说明，用于解释虚拟训练系统设计时应守住哪些任务边界。

它不是系统验收报告，也不是工程实现说明。

## 适合怎么用

- 用作虚拟训练系统立项、方案讨论或设计评审的背景材料。
- 帮助团队确认场景构建不是单纯建模展示，而是要服务训练目标。
- 作为后续实验设计、过程数据采集和专家评审的前置说明。

## 技术说明主线

- 先定义训练目标：训练谁，训练什么，训练到什么程度。
- 再定义场景对象：哪些空间、装备、危险源、角色和事件必须复现。
- 然后定义交互流程：观察、选择、操作、反馈、复盘是否闭环。
- 最后定义评价证据：任务时间、错误率、过程评分、记录完整性、系统稳定性等。

## 可保留内容

- 虚拟训练作为辅助训练工具的定位。
- 场景构建由训练目标牵引的原则。
- 展示模型与训练环境的区分。
- 运行性能、可重复性和可维护性也是训练性的组成部分。

## 需要补充的内容

- 如果转成正式技术说明，应补充具体系统对象、使用角色、训练流程和数据字段。
- 如果要支撑验收，应补充真实测试记录、操作日志、评价表和专家审阅意见。

## 不建议怎么用

- 不建议据此宣称系统已经达到生产可用。
- 不建议据此宣称训练效果、岗位能力或真实任务迁移已被验证。
- 不建议把文献整理工具链的检查结果等同于系统质量或论文质量验收。
