# 不可声明边界 v1.0

## 不可声明为论文终稿

- 不可声明当前稿件已达到最终论文质量。
- 不可声明题目、摘要、贡献 statement、结论和引用格式已经适配目标期刊、会议或课程要求。
- 不可声明参考文献已经完成目标规范下的最终核验。

## 不可声明训练效果已证明

- 不可声明虚拟训练已经显著提升真实任务能力。
- 不可声明虚拟训练已经提升协同表现、操作熟练度、安全表现或长期保持效果。
- 不可声明虚拟训练可以替代真实训练、实装操作或最终能力评价。
- 不可声明文献分析等同于实验评价、对照研究或专家验收。

## 不可声明系统或工具链已生产就绪

- 不可声明当前交付代表生产级 RAG 系统。
- 不可声明当前交付代表 broad RAG readiness。
- 不可声明当前交付具备 RuntimeAuthorization。
- 不可声明本地检查通过等同于系统验收、部署许可或安全授权。

## 不可声明已访问或验证的内容

- 不可声明已检查原始 PDF。
- 不可声明已检查原始 Obsidian 笔记正文。
- 不可声明已检查 raw chunks、source paths、vectors、FAISS binaries、WriteLab payloads 或 Zotero keys。
- 不可声明已访问 browser、CDP、cloud、MiniApp runtime、外部 RAG 服务或云向量数据库。
- 不可声明已获得新的 live resource access。

## 应避免的强表述

- 避免：“显著提升”“有效替代”“全面改善”“已经证明”“生产可用”“最终验收通过”。
- 避免：“系统已验证论文质量”“工具链已保证引用准确”“RAG 已完成广泛就绪”。
- 避免把“可交给人工审阅”说成“可直接正式提交”。
