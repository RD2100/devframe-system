# 参考文献与引用格式审计 v1.0

审计对象：

- `integration/artifacts/paper-drafts/local-paper-rag-clean-manuscript-v1.0.md`

## 结论

v0.9 中发现的主要格式问题是：正文首次引用顺序为 `1,3,4,6,2,5`，不符合顺序编码制的常规编号逻辑。

v1.0 已处理该问题：

- 原 `[1]` 保持为新 `[1]`。
- 原 `[3]` 调整为新 `[2]`。
- 原 `[4]` 调整为新 `[3]`。
- 原 `[6]` 调整为新 `[4]`。
- 原 `[2]` 调整为新 `[5]`。
- 原 `[5]` 调整为新 `[6]`。
- 参考文献表已按首次出现顺序重排。
- 正文相邻引用已压缩为 `[2-3]`、`[1-4,6]`、`[2,4,6]` 等紧凑形式。

因此，v1.0 已经达到“顺序编码制初步一致”的状态。

## 仍需人工核对

本审计未联网、未读取原始 PDF、未读取 Obsidian 原文、未访问外部数据库，因此不能确认 bibliographic truth。以下事项仍需人工核对：

1. 每条文献的作者、题名、期刊名、年份、卷、期、页码是否与权威来源一致。
2. DOI 是否准确存在。
3. 第 `[3]`、`[4]` 条中的“文章编号”是否需要保留。
4. 学位论文 `[6]` 是否需要补充保存地点，例如“长沙”，该地点不能由自动化凭空确认。
5. 如果目标模板要求上标引用，应在最终排版阶段统一处理。
6. 如果目标模板要求 3 名以上作者写“等”，需按模板调整第 `[5]` 条作者列表。

## 建议优先级

1. 如果只是今晚交付审阅稿，v1.0 当前引用状态可用。
2. 如果要正式投稿或提交，先确定目标格式，再逐条人工核对元数据。
3. 不建议自动补全未知地点、卷号、DOI 或作者缩写规则。

## 边界

本审计只处理格式和一致性，不证明参考文献内容真实支撑正文论断，也不构成最终引用验收。
