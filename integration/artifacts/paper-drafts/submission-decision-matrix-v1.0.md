# v1.0 用途与提交决策矩阵

## 当前稿件定位

`local-paper-rag-clean-manuscript-v1.0` 当前最稳妥的定位是：

- 短论文审阅稿；
- 技术札记审阅稿；
- 综述型课程论文审阅稿；
- 内部研究简报基线稿。

它还不是正式期刊最终投稿稿。

## 如何选择用途

| 用途 | 是否适合当前 v1.0 | 需要补什么 |
|---|---|---|
| 内部研究简报 | 适合 | 可直接附上用途说明和边界清单 |
| 课程论文 | 基本适合 | 按课程模板调格式，人工核对参考文献 |
| 技术札记 | 适合 | 弱化“研究证明”语气，突出系统边界和实践价值 |
| 短论文投稿前审阅 | 适合 | 继续做参考文献核对、模板排版、人工质量审阅 |
| 正式期刊投稿 | 暂不适合直接投 | 需要目标期刊模板、参考文献元数据核验、人工审阅 |
| 实证研究论文 | 不适合直接定位 | 需要实验设计、对照组、过程数据、统计分析 |

## 推荐决策

如果今晚目标是“有一份能拿出去给人看的成果”，选择：

```text
短论文审阅稿 / 技术札记审阅稿
```

如果目标是“立刻正式投稿”，当前还差：

```text
目标期刊模板 + 逐条参考文献核验 + 人工论文质量审阅
```

如果目标是“项目阶段验收”，选择：

```text
内部研究简报 + v1.0 交付包 + 非最终边界说明
```

## 不建议的定位

当前不建议把 v1.0 说成：

- 已完成投稿稿；
- 训练效果实证论文；
- 系统有效性证明论文；
- 生产可用系统验收报告；
- RAG 自动生成论文最终稿。

## 下一步动作

1. 先决定使用场景。
2. 若是审阅，直接发 DOCX 和 v1.0 package。
3. 若是课程提交，按课程模板做排版。
4. 若是投稿，先定期刊或会议模板，再做 v1.1。
