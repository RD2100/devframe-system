## 1. Purpose

This report extends the existing NEG-001 through NEG-030 governance fixtures
with parent integration cases needed by `devframe-system`.

It does not create fixtures or run tests. It defines the expected negative
behavior that future `test-frame`, `agent-acceptance`, or parent read-only
checks should enforce.

## 2. Existing Coverage

Existing negative acceptance tests already cover:

- fake green;
- blocked reported as pass;
- executor self-approval;
- missing evidence;
- forbidden Phase 0-5 actions;
- write scope violations;
- command injection;
- path traversal;
- dirty baseline mutation;
- missing pre/post git status.

Parent-specific gaps remain around integration promotion and pin readiness.
