- parent intake;
- evidence ledgering;
- submodule pin review;
- metadata-only report classification;
- synthetic/offline contract consumption;
- non-final readiness discussion.

## What This MVP Is Not

This MVP is not:

- final governance acceptance;
- production readiness;
- live paper-quality acceptance;
- complete source-level safety certification;
- control-plane runtime activation;
- CI/CD enforcement;
- canary/rollback execution;
- ZIP verifier runtime implementation;
- PDF, attachment, note, or full-text processing readiness;
- Obsidian, RAG, WriteLab, browser/CDP, cloud, or MiniApp readiness.

## Current Finality Rule

No child module, test suite, ZIP package, dry-run, metadata-only report, or
parent pin can issue final acceptance by itself.

Final acceptance remains unavailable until the parent has:

1. a final-verdict gate with independent reviewer provenance;
2. explicit resource-class accounting;
3. verified runtime authorization for any live resource;
4. source-level safety review for activated runtime paths;
5. end-to-end dry-run evidence that does not overclaim;
6. human approval for any final decision.
