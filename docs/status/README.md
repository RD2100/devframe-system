# Status Documents

This directory keeps operational status out of the GitHub root while preserving
the handoff trail for reviewers and agents.

| File | Purpose |
|---|---|
| `current-delivery.md` | Active reviewer-facing delivery handoff |
| `runbook.md` | Safe read-only checks and phase boundaries |
| `submodules.md` | Submodule roles and pinned revisions |
| `bootstrap-report.md` | Bootstrap result summary |
| `completion-matrix.md` | Phase completion matrix |
| `integration-status.md` | Integration status narrative |
| `paper-feature-status.md` | Paper feature delivery status |
| `risk-register.md` | Project risk register |

Keep product-facing explanation in the root README files. Keep process state,
handoff evidence, and temporary operational summaries here.
